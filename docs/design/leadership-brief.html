01 · 问题

合同要素抽取调优，业务面临三难

难点 1

诊断链路长

Badcase 要进 Trace 手工逐条归因，判断是召回、规则还是模型问题，耗时且依赖个人经验。
难点 2

调优门槛高

高度依赖长 Prompt 工程，业务人员难上手；改一处句式可能影响其他条款。
难点 3

回归难评估

改了规则，不知是否把原本对的改坏；缺版本管理与前后对比，调优凭感觉。
本质痛点：调一个条款，要在平台、外部大模型、Excel 之间反复复制粘贴、手改提示词、凭感觉回归。
02 · 方案

方案：构建一个独立的「规则实验室」

  • 测试集驱动生成：上传标注测试集，自动产出「语义 BOM」—— 业务定义 + 抽取规则 + 召回画像（关键词、语义查询、章节、正例）。
  • 错例智能归因优化：跑批后导入错例，自动 5 类归因，定向修订规则/画像。
  • 版本化回归：每版 BOM 快照 + 准确率前后对比（修复/回归/净收益），改好改差一目了然。
  • 解耦轻量：不直连抽取平台，靠导入导出桥接；不替代平台、不与质量运营重复。

04 · 工作原理

两阶段生成 + 自检 + 归因优化

阶段 1
语义定义 + 抽取规则
从测试集归纳条款本质与抽取边界（含拦截/匹配规则）。
阶段 2
召回画像（派生）
由定义派生关键词、语义查询、章节、正例，用于召回。
阶段 3
规则自检
验证正例能抽、反例能拦，抓自相矛盾与过拟合。
优化时对错例做 5 类归因，定向修复：
召回问题BOM 规则问题大模型推理问题Prompt 模板待优化混合问题
覆盖率阈值固定 80% 不可调 → 优化靠"让模型抽得更完整/更准"达标，不靠调阈值取巧。
05 · 可信度

不把 AI 草稿当事实

✓ 事实型（可直接信）

准确率、修复/回归数、版本对比 —— 来自平台跑分结果导入，确定性计算。

⚠ AI 产出（待人审）

定义、规则、画像、归因 —— 标注"待复核"，经人工确认 + 规则自检后才采纳。
  • 不预设"优良中差"评级：分类阈值待业务确认后才上线，工具只展示事实数据。
  • 可解释、可追溯、可回滚：每条规则有依据，每个版本可回滚。
06 · 价值

降门槛、可解释、解耦

减少 Prompt 工程

业务自助生成/优化规则，不再依赖手写长 Prompt。

业务自助调优

合同商务人员可自主维护条款规则，降低对技术依赖。

可解释可追溯

每条规则有依据，每次改动有记录，每版可回滚。

解耦不侵入

不改造平台，导入导出即用；不与质量运营重复。
