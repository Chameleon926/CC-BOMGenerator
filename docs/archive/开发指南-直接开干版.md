## 3. 你要写的东西（详细到文件名）

### 第一件事：`contracts/*.py`

```python
# src/cc_bom_generator/contracts/bom.py
from pydantic import BaseModel
from typing import List, Optional

class ExtractionRule(BaseModel):
    rule: str                    # 纯规则逻辑
    fixes: str = ""              # 修订依据（人审用，不录入平台）

class ExtractionRules(BaseModel):
    absolute_interception_rules: List[ExtractionRule] = []
    core_match_rules: List[ExtractionRule] = []

class RecallProfile(BaseModel):
    positive_keywords: List[str] = []
    confusion_words: List[str] = []
    section_hints: List[str] = []
    semantic_queries: List[str] = []
    positive_examples: List[str] = []

class BOM(BaseModel):
    clause: str
    block_code: str
    version: int = 1
    semantic_definition: str = ""
    extraction_rules: ExtractionRules = ExtractionRules()
    recall_profile: RecallProfile = RecallProfile()
```

```python
class Clause(Base):
    __tablename__ = "clause"
    block_code = Column(String(64), primary_key=True)
    block_name = Column(String(128), nullable=False)
    domain = Column(String(32))

class BomVersion(Base):
    __tablename__ = "bom_version"
    id = Column(Integer, primary_key=True, autoincrement=True)
    block_code = Column(String(64), ForeignKey("clause.block_code"))
    version = Column(Integer, nullable=False)
    bom_json = Column(JSON, nullable=False)      # BOM 的完整 JSON 快照
    parent_version = Column(Integer)
    source = Column(String(16))                  # "generate" | "optimize"
    created_by = Column(String(32))
    created_at = Column(DateTime, default=func.now())
```

### 5.1 杨力什么时候能开始写诊断？

等你的 `contracts/bom.py`、`contracts/diagnosis.py`、`contracts/trace.py` 提交后。因为 diagnose 要读写这些模型。

### 5.2 troubleshoot table

| 问题 | 做法 |
|---|---|
| **契约变了** | 改 contracts/ → commit → push。**不另通知，差异化 git log 自己看** |
| **表结构要改** | `alembic revision --autogenerate` → commit 迁移脚本 |
| **被对方改动阻塞** | 先 `git merge` 对方分支，解决冲突 |
| **有分歧** | 先按更保守的方式走（不改已有接口的情况下加字段），再当面聊 |
