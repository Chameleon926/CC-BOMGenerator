# 功能拆解 · 算法逻辑 / 输入依赖 / 可信度

> 目的：把平台每个功能拆细，说清**输出什么、怎么算、依赖什么输入、可信度多高**。
> 核心原则：**不把"AI 猜的"当"事实"示人；不预设未经领导认可的分级标准。**

## 可信度图例

| 标记 | 含义 | 对外态度 |
|---|---|---|
| 🟢 事实/计算型 | 确定性逻辑或平台导入的事实 | 可直接展示，无需复核 |
| 🟡 LLM 生成型 | 大模型产出，可能错 | **必须标注"AI 生成·待人工复核"**，不作为定论 |
| 🔴 阈值判断型 | 依赖人工定义的阈值（如"待优化线"） | **阈值未经领导定义前，不结论、不展示评级** |

## 逐功能明细

### 1. 数据加载与解析  🟢
- **输出**：标准化测试集（文档→语义块→语义项+期望值+上下文）。
- **算法**：pandas 读 xlsx/csv（中英列名自动映射）→ 按 `语义块编码/名称` 分组。
- **输入依赖**：用户上传的 Excel/CSV（列：期望值、语义块、文档ID…）。
- **可信度**：🟢 确定性解析，事实。

### 2. 候选近义去重  🟢
- **输出**：去重后的候选正例列表 + 被去掉项明细。
- **算法**：精确去重 → `difflib.SequenceMatcher` 字符相似度贪心（默认阈值 0.8）。
- **输入依赖**：期望值列表。
- **可信度**：🟢 确定性。⚠️ 字符相似度对中文偏粗（"违约金"≈"违约损失"可能误并），Phase 2 升级 embedding。

### 3. Stage1 定义+规则生成  🟡
- **输出**：语义定义（含/不含边界）+ 抽取规则（拦截/匹配）+ coverage_check。
- **算法**：LLM 从条款名+期望值归纳；optimize 时吃 badcase+trace 先诊断再改规则。
- **输入依赖**：条款名、期望值、（optimize）当前BOM、badcase、trace。
- **可信度**：🟡 **AI 生成，必须人审**。规则可能过拟合（写死公司/金额）或欠覆盖。已加程序化关键词过滤 + Stage3 自检兜底。

### 4. Stage2 召回画像(派生)  🟡
- **输出**：正向关键词/易混淆词/章节提示/语义查询/正例 + derivation_check。
- **算法**：LLM **从 Stage1 定义+规则派生**（关键词取自定义领域词、语义查询改写定义、易混淆词来自拦截规则）。
- **输入依赖**：Stage1 输出、候选正例。
- **可信度**：🟡 AI 生成，需人审。关键词已硬过滤（短/无数字/无整句）。

### 5. Stage3 规则自检  🟡
- **输出**：逐条验证（正例能否抽/反例能否拦）+ 红旗 + 覆盖估计 + 结论。
- **算法**：LLM 审核员拿BOM+正例+反例做验证（低温 0.0）。
- **输入依赖**：生成的BOM、候选正例、误抽反例。
- **可信度**：🟡 AI 验证，**是"第二意见"不是"绝对正确"**。能抓显性矛盾（如剔除规则误杀正例），但不保证全对。Phase 2 可加确定性规则引擎提升为🟢。

### 6. Badcase 诊断（5类归因）  🟡
- **输出**：每条 badcase 的 category（召回/混合/BOM/Prompt模板/推理）+ reason + suggested_fix + fix_target。
- **算法**：LLM 依据 trace 证据归因（原文窗口→召回、reasoning→推理、prompt I/O→模板）。
- **输入依赖**：badcase（期望/实际）、trace（可选，无则低置信）。
- **可信度**：🟡 **归因是 AI 判断，非事实**。无 trace 时只能判症状（误抽/漏抽），类别低置信。**必须人审后才作为优化依据**。

### 7. 数据脱敏  🟢(规则) / 🟡(NER)
- **输出**：脱敏文本 + 可逆映射表。
- **算法**：规则替换（手机号/身份证/金额→掩码或占位）+ NER（公司名/人名→角色占位）。
- **输入依赖**：原始文本/数据；脱敏规则配置。
- **可信度**：规则部分🟢确定性；NER 部分🟡（可能漏识别/误识别）。**出网前自动执行**，业务在工具内仍看真实数据。

### 8. 版本管理+Diff  🟢
- **输出**：版本快照、版本间字段 diff、准确率演进。
- **算法**：每次生成/优化存快照（SQLite+JSON）；diff = 字段级对比。
- **输入依赖**：历史 BOM 快照。
- **可信度**：🟢 确定性存储与对比。

### 9. 优化收益/回归计算  🟢
- **输出**：本次优化的 修复N / 回归M / 净收益 / 回归守护（改A是否坏B）。
- **算法**：版本N vs N-1 两份**导入的跑批结果**做 badcase 集合 diff：N-1错→N对=修复；N-1对→N错=回归。
- **输入依赖**：**两版跑批结果 Excel（从平台导入）**。无导入则无法算。
- **可信度**：🟢 确定性 diff，**前提是导入的跑批结果准确**。

### 10. 大盘：准确率/趋势/归因占比  🟢(+🟡)
- **输出**：整体准确率、趋势、badcase 5类归因占比、风险排序。
- **算法**：准确率/趋势=**导入跑批结果的事实聚合**（🟢）；归因占比=**依赖功能6的AI归因**（🟡）。
- **输入依赖**：跑批结果（导入）+ 诊断结果。
- **可信度**：准确率部分🟢；归因占比🟡（继承诊断的不确定性）。
- ⚠️ **"重点关注=准确率最低/下降最多"是事实排序，不是评级**。

### 11. 测试集分析与建议  🟡
- **输出**：测试集问题（数据不均/缺场景）+ 补样本建议。
- **算法**：LLM 分析测试集分布（仅当某条款准确率低时触发）。
- **输入依赖**：测试集、准确率（判断是否低）。
- **可信度**：🟡 AI 建议，参考用。**仅低准触发**，不做常态展示。

### 12. ~~条款健康度分级~~  🔴 已移除
- **原设计**：按准确率分优良中差（🟩🟨🟥）。
- **问题**：优良中差的**阈值标准未经领导认可与检验**，属于自造标准。
- **处置**：**移除整体健康度评级**。所有"待优化/风险/已优化"之类的分级，**阈值必须由业务领导定义并检验后才上线**；在此之前只展示**事实数据**（准确率、趋势、归因占比）与**事实排序**，不做主观评级。

---

## 给业务/领导的一句话总结

> 这个工具里，**准确率、修复/回归数、版本diff 是"算出来的事实"（可信）**；**定义/规则/画像/归因/建议 是"AI 生成的草稿"（必须人审）**；**任何"优良中差/待优化"的分级，标准没定之前一律不展示**——避免拿 AI 的猜测和自造的阈值去误导决策。
