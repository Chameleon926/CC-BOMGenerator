## 1. 生成场景总览

### 1.1 一句话

用户导入合同测试集 Excel → 数据清洗（分类、去重）→ 基于清洗后的数据生成语义 BOM（定义 + 抽取规则 + 召回画像）→ 组装成旧平台可用的完整提示词。

### 1.2 数据流（两个模块的交接点）

```
用户上传 Excel
     │
     ▼
┌─────────────────────────────────┐
│  A 模块（杨力）数据预处理          │
│  ├─ 解析 Excel（中英列名自适应）    │
│  ├─ 按条款分组                     │
│  ├─ 精确去重 + 近义去重             │
│  └─ 输出：CleanedTestSet           │
└──────────────┬──────────────────┘
               │  交接契约：CleanedTestSet
               ▼
┌─────────────────────────────────┐
│  B 模块（林大宇）核心生成           │
│  ├─ B1 召回画像（关键词/混淆词等）   │  ← 部分用统计/NLP，部分用大模型
│  ├─ B2 语义定义 + 抽取规则          │  ← 大模型
│  ├─ B3 规则自检                     │  ← 大模型
│  └─ B4 完整提示词组装               │  ← 模板拼装（不用大模型）
│  输出：BOM + FullPrompt             │
└──────────────────────────────────┘
```

## 4. 算法与实现建议（重点）

### 4.1 总原则：能不用大模型就不用

大模型用于"需要语义理解和归纳"的环节（语义定义、抽取规则）。能用统计和 NLP 搞定的（关键词、混淆词、格式模板），**用传统方法更快、更稳定、更可解释**。

### 4.2 每个环节的方法选型

| 环节 | 方法 | 是否用大模型 | 理由 |
|---|---|---|---|
| **正向关键词** | TF-IDF + 词频统计 + 规则过滤 | ❌ 不用 | 关键词是高频通用词，统计就够 |
| **易混淆词** | 基于关键词的对比排除 + 领域词典 | ❌ 不用 | 与正向词互斥的词 |
| **章节提示** | 从测试集的 sectionPath 字段统计 | ❌ 不用 | 如果有章节信息直接统计 |
| **语义查询句** | 基于定义改写（模板化） | ⚠️ 轻量调 | 拿定义做模板变换 |
| **语义定义** | 大模型归纳 | ✅ 用 | 需要语义理解和泛化 |
| **抽取规则** | 大模型生成 | ✅ 用 | 需要逻辑推理 |
| **正例挑选** | 聚类 + 中心点选择 | ❌ 不用 | 多样性挑选用聚类 |
| **提示词组装** | 字符串模板拼接 | ❌ 不用 | 纯拼装 |

### 4.3 关键词抽取的具体算法（B 模块核心，不用大模型）

这是你问的重点。分三步：

#### 第一步：分词 + 词频统计

```python
import jieba
from collections import Counter

def extract_candidate_keywords(values: List[str], top_n: int = 30) -> List[tuple]:
    """
    从期望值列表中抽取候选关键词。
    返回 [(词, 得分), ...]
    """
    # 1. jieba 分词（合同文本用 custom_dict 增强领域词）
    all_words = []
    for val in values:
        words = jieba.lcut(val)
        all_words.extend(words)

    # 2. 过滤停用词 + 标点 + 单字
    filtered = [w for w in all_words
                if len(w) >= 2
                and w not in STOPWORDS
                and not w.isdigit()
                and not all(c in '，。、：；！？（）()[]【】' for c in w)]

    # 3. 词频统计
    freq = Counter(filtered)

    # 4. TF-IDF 加权（如果有多个条款的语料做 IDF）
    # 简化版：直接用词频 / 总词数 作为 TF
    total = sum(freq.values())
    scored = [(word, count / total) for word, count in freq.most_common(top_n)]
    return scored
```

#### 第二步：规则过滤（防过拟合）

```python
def filter_keywords(candidates: List[tuple]) -> List[str]:
    """
    过滤掉不适合做精确匹配的候选词。
    """
    result = []
    for word, score in candidates:
        # 规则1：长度 2-8 字（中文）
        if len(word) < 2 or len(word) > 8:
            continue
        # 规则2：不含数字/金额/百分比（防过拟合）
        if any(c.isdigit() or c in '%％万元亿' for c in word):
            continue
        # 规则3：不含公司名/人名（用 NER 或正则）
        if re.search(r'[有限公司股份集团]', word):
            continue
        # 规则4：频次 ≥ 2（至少出现两次才算通用）
        if score < 2 / len(candidates):
            continue
        result.append(word)
    return result[:10]  # 取 top 10
```

#### 第三步：可选升级（小模型 / TF-IDF 加权）

如果词频统计不够好（比如"合同"这种通用词排第一），升级方案：

| 升级方案 | 方法 | 适用场景 | 复杂度 |
|---|---|---|---|
| **TF-IDF** | 把所有条款的期望值当文档集，算 IDF 权重 | "合同"这种通用词 IDF 低，自动降权 | 低（sklearn 一行） |
| **TextRank** | 基于词共现图做 PageRank 排序 | 抽出的词更"关键" | 中（jieba 自带） |
| **KeyBERT** | 用 BERT embedding 做相似度排序 | 抽出的词语义更准 | 中（需装 sentence-transformers） |
| **领域词典** | 预定义合同高频术语表，与统计结果取交集 | 精确、可控 | 低 |

**我的建议**：先用 jieba + 词频 + 规则过滤（零依赖、快、可解释），跑一轮看效果。如果关键词质量不够（比如出现太多"甲乙双方"这种噪声），再上 TF-IDF。**不建议一开始就上 BERT/KeyBERT，那是 Phase 2 的事。**

### 4.4 易混淆词的抽取方法

```python
def extract_confusion_words(keywords: List[str], all_values: List[str]) -> List[str]:
    """
    从期望值中找"与关键词相近但不是关键词"的词。
    方法：分词后，找那些与关键词字面相似（编辑距离≤2）
    但本身不在关键词列表里的高频词。
    """
    confusion = []
    kw_set = set(keywords)
    all_words = [w for val in all_values for w in jieba.lcut(val) if len(w) >= 2]
    freq = Counter(all_words)

    for word, count in freq.most_common(100):
        if word in kw_set:
            continue
        # 与任一关键词编辑距离 ≤ 2 → 可能是混淆词
        for kw in keywords:
            if levenshtein(word, kw) <= 2:
                confusion.append(word)
                break
    return confusion[:6]
```

### 4.5 正例挑选（多样性聚类）

```python
from sklearn.cluster import KMeans
from sklearn.feature_extraction.text import TfidfVectorizer

def select_diverse_examples(values: List[str], n: int = 5) -> List[str]:
    """
    从期望值列表中选 n 个互相差异最大的代表性样本。
    方法：TF-IDF 向量化 → KMeans 聚类 → 每类取离中心最近的样本。
    """
    if len(values) <= n:
        return values

    vectorizer = TfidfVectorizer(tokenizer=jieba.lcut)
    X = vectorizer.fit_transform(values)
    k = min(n, len(values))
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    km.fit(X)

    selected = []
    for i in range(k):
        cluster_indices = [j for j, label in enumerate(km.labels_) if label == i]
        if not cluster_indices:
            continue
        # 取离中心最近的
        distances = km.transform(X[cluster_indices])[:, i]
        nearest = cluster_indices[distances.argmin()]
        selected.append(values[nearest])
    return selected[:n]
```

### 4.6 章节提示

```python
def extract_section_hints(testset, top_n=6):
    """直接从数据中统计章节名，不需要大模型。"""
    sections = [doc.section for doc in testset.documents if doc.section]
    freq = Counter(sections)
    return [s for s, _ in freq.most_common(top_n)]
```

```python
def assemble_full_prompt(bom: BOM) -> FullPrompt:
    """
    基于 BOM + 画像组装旧平台可用的完整提示词。
    纯模板拼装，不调大模型。
    """
    template = """你是一个专业的合同条款抽取助手。请从以下合同原文窗口中抽取语义块：{clause}（{block_code}）。

目标定义：{definition}

【抽取规则】
{rules}

【目标画像】
{profile}

--- 合同原文窗口 ---
（用户在此粘贴合同原文）

--- 输出格式 ---
请以 JSON 格式返回抽取结果。"""

    rules_text = format_rules(bom.extraction_rules)
    profile_text = format_profile(bom.recall_profile)

    prompt = template.format(
        clause=bom.clause,
        block_code=bom.block_code,
        definition=bom.semantic_definition,
        rules=rules_text,
        profile=profile_text,
    )
    return FullPrompt(clause=bom.clause, prompt_text=prompt, bom_snapshot=bom,
                      assembled_from=["定义", "规则", "画像"])
```

```
  │     ├─ 语义查询   ← 大模型（基于定义改写）
  │     └─ 正例       ← 聚类挑选（不用大模型）
  │
  ├─ B3. generate.py           定义+规则（大模型 Stage1）
  │     └─ 输出：BOM 的 definition + extraction_rules
  │
  ├─ B4. verify.py             规则自检（大模型 Stage3）
```

## 8. 新平台抽取工作流备忘（B 模块准确率保障依据）

> 林大宇负责算法编排和准确率，最终生成的 BOM + 画像要拿到新平台跑验证。
> 以下是新平台的抽取链路，B 模块生成的每个字段必须匹配新平台的消费方式。

### 8.1 新平台完整抽取链路

```
文档块加载(doc_chunk_loader)
  → 召回画像匹配（关键词精确匹配 + 语义查询向量召回）
    → 语义块混合chunk召回器（分段覆盖率匹配：单段80% + 整体80%）
      → LLM 抽取（用定义+规则组装的 Prompt）
        → 覆盖率判定（实际抽取 覆盖 期望 ≥ 80% → 匹配）
```

### 8.2 B 模块输出 → 新平台消费对照表

| B 模块产出 | 新平台怎么用 | 准确率关键点 |
|---|---|---|
| **正向关键词**（positive_keywords） | **精确匹配**合同原文片段——原文里必须出现这个词才算命中召回 | 必须短词（2-8字），太长原文不会有=过拟合=召回失败 |
| **语义查询**（semantic_queries） | 作为**向量召回**的查询句，在文档 chunk 里做语义相似度检索 | 要能准确描述"这个条款指什么"，不是关键词堆砌 |
| **章节提示**（section_hints） | 提示系统该条款大概率出现在哪些章节，缩小召回范围 | 要写真实章节名（如"付款条款"），不要编造 |
| **易混淆词**（confusion_words） | 标记为噪声词，召回时降权或排除 | 要写**容易和本条款搞混**的词，不是随便找几个 |
| **正例**（positive_examples） | **新平台用于向量召回锚点**（不是进 Prompt） | 要有代表性、多样性，覆盖条款的不同表达方式 |
| **语义定义**（semantic_definition） | 拼入 LLM 抽取 Prompt 的"目标定义"段 | 要泛化（不含公司名/金额），让模型在任意合同上都能用 |
| **抽取规则**（extraction_rules） | 拼入 LLM 抽取 Prompt 的"抽取规则"段 | 拦截规则防误抽、匹配规则防漏抽 |
| **完整提示词**（FullPrompt） | 直接粘贴到**旧平台**使用（适配旧平台） | 基于 BOM+画像+示例模板组装 |

### 8.3 准确率保障的 5 条铁律

1. **关键词必须短**——精确匹配用，原文没有的词=召回失败。程序化过滤含数字/超长/整句。
2. **语义查询必须像定义**——它是定义的改写句，给向量召回用。不是关键词列表。
3. **定义必须泛化**——不能写死公司名/金额/日期，否则换个合同就失效。
4. **规则不能自相矛盾**——拦截规则和匹配规则不能有重叠的触发条件。Stage3 自检要抓这个。
5. **正例必须有多样性**——100 条一样的只留 1 条代表性样本。聚类挑选。

### 8.4 新平台的覆盖率机制（影响 B 模块规则设计）

新平台判定"匹配"用**分段覆盖率**：
- 把期望值分成若干段
- 每段：实际抽取匹配长度 / 段长度 ≥ 80% 才计入分子
- 整体：所有有效段匹配长度之和 / 期望总长度 ≥ 80% 才判定匹配

**对 B 模块的影响**：生成的抽取规则必须让模型抽出**完整的期望 span**，不能只抽一部分。如果规则写太窄，模型只抽了表格没抽文字段落 → 覆盖率 < 80% → 判定漏抽。

**所以**：匹配规则要覆盖条款的所有组成部分（定义+表格+排程），拦截规则只拦确实不属于的。

### 9.4 设计约束

- **所有 FK 加 `ON DELETE RESTRICT`**，禁止级联删（防止孤儿记录）
- **所有 `created_at` 用 `server_default=func.now()`**（非 Python default），保证裸 SQL 也安全
- **表名统一复数**（clauses/bom_versions/pipeline_runs/...）
- **避开 MySQL 保留字**：source→bom_source, status→bom_status/run_status, model→model_name, sequence→seq
- **枚举字段用 SAEnum**，不用裸 String
- **唯一约束**：`(block_code, version)` on bom_versions, `(block_code, item_code)` on clause_items
