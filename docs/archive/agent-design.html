条款工作台
对话 Copilot 设计

一个挂在条款工作台右侧的"加速器"Agent：用自然语言查 badcase/归因、解释、生成/优化规则。复用三阶段引擎，不重写；生成必经人审，回答必带证据与可信度。

版本V0.1定位加速器（非主线）架构LangGraph 状态机+人审上线Phase 2

01定位

Copilot 是"加速器"，不是"主线"。主线是条款工作台（三阶段）+ 质量大盘。

  • 不替代工作台：核心生成/优化/版本仍在工作台；Copilot 只是把高频操作"说一句话就能做"。
  • 后置上线：三阶段 + 工作台 + 大盘跑顺后才上 Copilot（它本质是这三者的"对话式外壳"）。
  • 面向三类用户：合同商务部（操作条款）、数据专家（看归因/回归）、领导（看收益/风险）。
一句话

Copilot = 三阶段引擎 + 一组查询/执行工具 + 一个带人审断点的对话循环。它编排，不重写核心逻辑。

03能力清单（5 类）

  1. 查询：badcase / 当前 BOM / 归因 / 版本 / 大盘指标。
  2. 解释：为什么准/不准（引用归因、trace、版本 diff）。
  3. 生成/优化：跑三阶段（generate/optimize），产出 BOM 草稿。
  4. 批量编排：对多个低准确率条款批量生成建议。
  5. 新手引导："我该怎么做？" → 分步指引。

04形态设计

挂在工作台右侧/悬浮窗。主区仍是工作台；Copilot 不喧宾夺主。

4.1 消息类型（不止文字）

  • 文本：自然语言回答 + 结论。
  • 卡片：badcase 列表、规则建议、小图表（归因占比/趋势）。
  • 操作按钮：采纳 / 去优化 / 查看证据 / 跳转条款。
  • 证据 + 可信度：每条 AI 回答底部标注"依据：badcase#X / trace"与 🟡AI生成·待复核。

05架构（LangGraph 状态机 + 人审断点）

# 对话循环（每轮） 用户输入 │ ▼ [意图识别] ── 路由 ──▶ [查询类] ──▶ 选工具(query_*) ──▶ 作答(带证据) │ └──▶ [生成/优化类] ──▶ run_generate/optimize(复用三阶段) │ 产出"建议改动(草稿)" │ ⏸ 人审断点（interrupt） # 用户确认才落库 │ [采纳] → apply(版本化) → 作答 [拒绝/调整] → 回到生成
  • 状态：当前选中条款（上下文）+ 对话历史 + 短期记忆。
  • 复用引擎：generate/optimize/verify 节点直接调用，Copilot 不重写核心。
  • 人审断点：LangGraph interrupt，生成草稿后暂停，用户点"采纳"才 apply 落库——Agent 不擅自改规则。
  • 选型：LangGraph（状态机+人审，最贴合）；备选 Anthropic Agent SDK / Dify·Coze（低代码）。

06工具集契约（Agent 可调用的工具）

工具输入输出可信度
query_badcasesblock_code, type, limitbadcase 列表🟢 事实
query_clause_bomblock_code, versionBOM 快照🟢 事实
query_diagnosisblock_code, version归因结果🟡 AI 归因
query_dashboard指标名/范围准确率/收益/归因占比🟢/🟡
explain_version_diffblock_code, v_a, v_b版本差异说明🟢 diff+🟡 解读
run_generate / run_optimizeblock_code, 数据BOM 草稿（待审）🟡 AI 生成
apply_bomblock_code, 草稿, 确认新版本（落库）🟢 仅人审后
工具即节点

查询工具直接读库（事实🟢）；run_* 复用三阶段节点（AI🟡）；apply 仅在人审确认后执行。Agent 本身不持有业务逻辑。

08记忆设计

  • 短期记忆：当前选中条款（上下文）+ 本会话对话历史。会话结束即清。
  • 长期记忆（Phase 2）：把"trace→成功修法"对子向量化入库；Agent 遇相似 badcase 时检索借鉴（few-shot）。这是向量库的正当用途之一。
  • 不跨用户：记忆按用户/项目隔离，不泄露他人数据。

09安全与铁律

铁律落地
不擅自改规则生成后停在人审断点，用户确认才 apply
回答带证据+可信度每条 AI 回答标注依据(引用 badcase/trace/版本) + 🟡待复核
复用三阶段引擎Copilot 只编排+对话，核心逻辑调用现有节点，不重写
出网脱敏调外部 LLM 前自动脱敏（同主流程）
不预设评级被问"健康度/优良中差"时，回答"阈值待业务定义"，不编标准

10分期

期Copilot 范围
Phase 1不做（先把工作台+三阶段+大盘跑顺）
Phase 2查询类 + 解释类 + 生成/优化(带人审)；短期记忆；侧边窗形态
Phase 3长期记忆(向量库 few-shot)、批量编排、多用户、与平台对接
前提

Copilot 的所有能力都依赖底层节点 + 数据（导入的跑批结果/诊断/版本）。底层没数据/没跑顺，Copilot 就是无源之水。所以必须先 Phase 1。
